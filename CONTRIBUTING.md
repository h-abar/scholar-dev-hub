# Contribution Guidelines

Thank you for considering contributing to **Scholar Dev Hub**! 🎉

سواء كنت تضيف موردًا جديدًا أو تصلح رابطًا أو تحسّن وصفًا — كل مساهمة مرحّب بها.

## How to Contribute

1. **Fork** this repository.
2. Create a branch: `git checkout -b add/resource-name`
3. Add your resource following the format below.
4. Commit with a clear message: `Add: Resource Name`
5. Open a **Pull Request** and fill in the PR template.

## Entry Format

Every entry must follow this exact format:

```markdown
- [Name](https://example.com) - One short sentence describing why it's valuable, ending with a period.
```

- Use the **official name** and a **direct link** (no URL shorteners, no affiliate links).
- Place the entry in the **most relevant section**, in **alphabetical order** within its subsection.
- Descriptions must be **one line**, factual, and free of marketing language ("the best", "amazing").

## Quality Criteria

A resource is accepted if it:

- ✅ Is **free**, **open-source**, or has a genuinely useful **free tier**.
- ✅ Is **actively maintained** (not abandoned for years).
- ✅ Is **high quality** and useful for developers, students, academics, or researchers.
- ✅ Respects others — no pirated content, no paywall-bypassing tools (legal OA finders like Unpaywall are fine).

## Rules

- 🔍 **Search first** — make sure the resource is not already listed (no duplicates).
- 🔗 **Check the link** — it must work over HTTPS.
- 1️⃣ **One resource per PR** — this makes review fast and easy.
- 🌐 Arabic and English resources are both welcome; Arabic resources go to the **الموارد العربية** section.
- ❌ Do not add your own project unless it is genuinely established and useful (self-promotion PRs will be closed).
- 🐛 Found a dead link? Open an issue using the **Dead Link** template, or send a fix PR directly.

## Suggesting Without a PR

Not comfortable with Git? No problem — open an issue using the **Suggest a Resource** template and we'll review it.

## Code of Conduct

Be kind and respectful. We're building a resource for learners and researchers everywhere — harassment or spam of any kind is not tolerated.
