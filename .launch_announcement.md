# 🎉 Scholar Dev Hub v1.1.0 is live — from curated list to a real clone-build workspace

> Building with AI is no longer about choosing the "best" coding agent. It's about creating an ecosystem where every AI tool works together.

**Scholar Dev Hub** is now more than a curated index — it's a reusable AI development environment.

## What's inside

- 🚀 **AI Coding Agents** — Claude Code, OpenCode, Cursor, **Gemini CLI**, **Kimi CLI**, **Codex**, **Qwen Code**, Aider, Cline, Windsurf, Continue, Void, Zed, Cody, Amp
- 🧠 **Optimized rules & prompts** — `rules/AGENTS.md`, `rules/CLAUDE.md` (read automatically by every modern agent)
- 🔌 **MCP integrations** — `rules/mcp-example.json` drop-in `.mcp.json` + the full MCP ecosystem mapped out
- ⚡ **Reproducible dev environment** — `.devcontainer/` for one-click VS Code / Codespaces (Python 3.12, Node 20, Git, uv, ripgrep)
- 📚 **Practical resources** — plus curated lists for students, professors, academics, and researchers, with a dedicated **Arabic resources** section

## The workflow

```bash
git clone https://github.com/h-abar/scholar-dev-hub.git
# Reopen in Container (VS Code / Codespaces), OR:
cp scholar-dev-hub/rules/AGENTS.md        your-project/
cp scholar-dev-hub/rules/CLAUDE.md        your-project/
cp scholar-dev-hub/rules/opencode.json    your-project/opencode.json
cp scholar-dev-hub/rules/mcp-example.json your-project/.mcp.json
# → Configure. Build.
```

As AI agents become part of every developer's workflow, the advantage will no longer come from using a model. It will come from the **quality of the engineering environment surrounding it**. This is my contribution toward making that environment open, reusable, and continuously evolving.
